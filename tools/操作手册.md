# 工具集操作手册

`tools/dist/` 包含三个独立可执行文件，用于帕斯特之梦 → 帕斯特之梦: 重生 的 ID 映射管理及 NBT 结构转换。运行时需保持在项目目录结构中（exe 通过相对路径定位 `document/design/` 下的文件）。

---

## 一、generate_id_mapping.exe — 生成映射文件

从 `document/design/ID映射表.md` 解析映射关系，生成 `document/design/mapping.json`。

```bash
./tools/dist/generate_id_mapping.exe
```

- 自动保留 `mapping.json` 中手动维护的节（如 `block_properties`），不会被覆盖
- 用法同 `python tools/generate_id_mapping.py`

---

## 二、lookup_id.exe — 查询映射

从 `document/design/mapping.json` 查询旧 ID 对应的新 ID。

```bash
./tools/dist/lookup_id.exe flower_16            # 单个查询
./tools/dist/lookup_id.exe flower_1 flower_2    # 批量查询
./tools/dist/lookup_id.exe -a                   # 列出全部映射
./tools/dist/lookup_id.exe -f list.txt          # 从文件批量查询（每行一个ID）
```

---

## 三、convert_structure.exe — NBT 结构方块转换

将旧模组 `.nbt` 结构文件中的方块 ID 按 `mapping.json` 转换为新 ID。原版 `minecraft:` 方块不受影响。

### 方式 A：交互模式（推荐）

将 `convert_structure.exe` 和 `mapping.json` 复制到 `.nbt` 文件所在的目录，双击运行：

```
工作目录/
  convert_structure.exe
  mapping.json
  pinkagaric_house_0.nbt
  dream_church_0.nbt
  converted/              ← 自动创建，输出到这里
```

1. 启动后列出所有 `.nbt` 文件（含文件大小和已转换标记）
2. 输入数字选择文件
3. 预扫描显示未映射和待搬运的方块，询问是否替换为空气 `[y/N]`
4. 转换完成，输出到 `converted/` 子目录
5. 返回文件列表，可继续转换或 `q` 退出

### 方式 B：命令行模式

```bash
./tools/dist/convert_structure.exe -i input.nbt -o output.nbt
./tools/dist/convert_structure.exe -i input.nbt --delete-unmapped   # 未映射方块替换为空气
```

### 转换规则

| 方块类型 | 行为 |
|----------|------|
| `minecraft:` 原版方块 | 忽略，不变 |
| mapping 中有映射 | 替换为新 ID，保留 Properties |
| mapping 中标记 `删除` | 替换为 `minecraft:air` |
| mapping 中标记 `待搬运` | 保留原样，提示警告 |
| mapping 中找不到 | 保留原样，提示警告（交互模式可自选替换为空气） |

### 属性覆盖

`mapping.json` 中的 `block_properties` 节用于在转换时强制写入方块属性。例如旧作物方块原作中没有生长阶段，统一方块后需区分成熟/未成熟：

```json
"block_properties": {
  "crop_0a": {"age": "1"},
  "crop_0b": {"age": "0"}
}
```

`crop_0a`（成熟植株，世界生成用）→ 转换后自动获得 `age=1`；`crop_0b`（生长阶段）→ `age=0`。

### 输出信息

```
  已转换 6 种方块类型（共 7 个 palette 条目）：
    crop_0a -> dyedream_corolla_crop [1次]
    crop_3a -> cloud_crop [1次]
  [!] 提示：2 个模组方块未在 mapping 中找到，保持原样：
    - meltdream_chest
    - vine_0
  已合并 1 个重复的 palette 条目。
```

## 四、mapping.json 结构

```json
{
  "blocks": {
    "旧方块短名": "新方块短名",
    "claypan_1": "删除",
    "biome_shadow_0": "待搬运"
  },
  "block_properties": {
    "crop_0a": {"age": "1"}
  },
  "items": { ... },
  "entities": {},
  "dimensions": {},
  "biomes": {},
  "effects": {},
  "features": {}
}
```

- `"删除"` — 该方块将被替换为空气
- `"待搬运"` — 暂不处理，转换时提示警告
- `block_properties` — 手动维护，`generate_id_mapping` 重新生成时自动保留

---

## 五、重新编译

修改 `tools/` 下的 `.py` 源码后：

```bash
pyinstaller --onefile --name convert_structure   --distpath tools/dist tools/convert_structure.py
pyinstaller --onefile --name generate_id_mapping --distpath tools/dist tools/generate_id_mapping.py
pyinstaller --onefile --name lookup_id           --distpath tools/dist tools/lookup_id.py
```
