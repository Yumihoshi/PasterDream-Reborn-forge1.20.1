# 源项目结构分析

> PasterDream 旧版模组（MCR 项目）—— 目录结构与模块划分说明  
> 源项目分为两大部分：`NOT_MODIFY/reference/`（Java 源代码）和 `NOT_MODIFY/reference-resources/`（资源与数据文件）。⚠️ 此目录内文件严禁修改。

---

## 一、源代码目录 `NOT_MODIFY/reference/`

源码根包：`net.pasterdream`，总计 **1814 个 Java 文件**，分布在以下模块中。

### 1.1 根包 `net/pasterdream/`（11 文件）

主类入口、杂项工具类。包括 BossBar 等顶层类。

---

### 1.2 `block/` — 方块模块（334 文件）

方块定义是旧项目中最大的模块之一。每个方块几乎都有一个独立的类文件。

| 子目录 | 文件数 | 说明 |
|--------|--------|------|
| `block/`（根） | ~200 | 方块类，每个方块一个文件。命名风格混乱，大量 `*BlockBlock.java` 冗余后缀，数字编号变体（如 `Flower1Block.java` ~ `Flower18Block.java`、`Grass1Block.java` ~ `Grass15Block.java`） |
| `block/display/` | 35 | 方块物品的显示类（`*DisplayItem.java`），用于自定义方块在物品栏中的渲染 |
| `block/entity/` | 79 | 方块实体（TileEntity），含大量 `StructureBlock*BlockEntity.java`（0~23，共 24 个结构方块实体）、功能方块实体（熔炉、箱子、祭坛等） |
| `block/model/` | 70 | 方块的自定义模型类（`*BlockModel.java` + `*DisplayModel.java`），对应每个有自定义模型的方块 |
| `block/renderer/` | 70 | 方块实体渲染器（BER），与 `block/entity/` 中的方块实体配对 |
| `block/listener/` | 1 | 客户端事件监听器（`ClientListener.java`） |

**主要问题**：
- 大量方块有独立的客制化类，即使不需要特殊逻辑（仅设置属性）
- 花草类方块使用数字编号（`Flower1` ~ `Flower18`、`Grass1` ~ `Grass15`），缺乏语义化命名
- 类名经常出现重复后缀（`*BlockBlock.java`）

| 子目录 | 文件数 | 说明 |
|--------|--------|------|
| `item/`（根） | ~280 | 物品类，每个物品一个文件。包括卡牌（`CalleCard0~9`）、饰品（`*CurioItem`）、材料、工具、武器等 |
| `item/inventory/` | 17 | 自定义物品容器/背包类 |
| `item/model/` | 5 | 物品的自定义模型类 |

**主要问题**：
- 与方块类似，大量不必要的独立类（仅用于注册普通物品）
- 数字编号变体（`Brokennotes0Item`、`CalleCard0~9`）

### 1.7 `procedures/` — 过程逻辑模块（**524 文件**，最大模块）

这是 **MCreator 自动生成** 的代码。每个文件对应一个 "Procedure"（过程），是 MCreator 可视化编程的产物。命名模式为 `{触发者/目标}_{行为}Procedure.java`，例如：
- `AaroncosLefthandDeathProcedure.java` — Aaroncos 左手死亡时触发
- `AaroncosLefthandSkillHitProcedure.java` — Aaroncos 左手技能命中时触发
- `DyedreamGrassBlockPlantProcedure.java` — 染梦草方块种植逻辑

**特点**：
- 全部由 MCreator 自动生成，代码风格高度统一
- 方法签名通常为 `public static void execute(LevelAccessor world, double x, double y, double z, Entity entity)`
- 包含大量硬编码坐标和数值
- 是搬运时最需要重写的部分 —— 需要用规范的 Java 代码重新实现逻辑

## 三、模块依赖关系总览

```
┌─────────────┐
│   init/     │  注册中心 —— 引用所有其他模块
│  (34 files) │
└──────┬──────┘
       │ 注册
┌──────┴──────────────────────────────────┐
│  block/  ──  block/entity/  ──  block/renderer/
│  item/   ──  item/inventory/ ──  item/renderer/
│  entity/ ──  entity/model/  ──  entity/layer/
│  fluid/     potion/     enchantment/
└──────────────────────────────────────────┘
       │ 逻辑驱动
┌──────┴──────────────┐
│  procedures/        │  MCreator 过程代码 —— 最大模块
│  (524 files)        │  包含所有交互逻辑、事件响应
└─────────────────────┘
       │ 运行时
┌──────┴──────────────────────────────┐
│  client/     network/    event/     │
│  jei/        command/    mixin/     │
│  world/      recipes/    utils/     │
└─────────────────────────────────────┘
```

## 四、与重制项目的对应关系

| 旧项目包 (`net.pasterdream`) | 新项目包 (`com.pasterdream.pasterdreammod`) |
|------------------------------|---------------------------------------------|
| `init/` | 分散到各模块包下（`world/level/block/ModBlocks.java`、`world/item/ModItems.java` 等） |
| `block/` | `world/level/block/`（无特殊逻辑用 `new Block()`，有特殊逻辑放 `custom/`） |
| `block/entity/` | `world/level/block/entity/`（BlockEntity） |
| `block/renderer/` | `client/renderer/block/`（BER） |
| `block/display/` | 数据生成器 + 模型 JSON 替代（不再需要自定义 DisplayItem 类） |
| `block/model/` | `client/model/block/` 或数据生成器替代 |
| `item/` | `world/item/` + `world/item/custom/` |
| `entity/` | `entity/`（含 `model/`、`layer/`、`renderer/` 子包） |
| `client/` | `client/` |
| `potions/` | `world/effect/` |
| `network/` | `network/` |
| `worldgen/` → `procedures/` | `worldgen/`（`ModConfiguredFeatures`、`ModPlacedFeatures`）+ 数据生成器 |
| `procedures/` | 手动重写为规范的 Java 业务逻辑，分散到各相关模块 |
| `jei/` | `jei/` |
