## 六、架构原则总结

1. **客制化类最小化**：无特殊机制的方块/物品直接用原版类，仅特殊逻辑的放在 `custom/` 子包
2. **模块分散注册**：注册中心按模块放置（如 `ModBlocks` 在 `world/level/block/`），而非集中在 `init/` 包
3. **数据生成器驱动**：所有可自动产出的 JSON 均由数据生成器生成，不手动编写
4. **命名规范严格**：ID 语义化 snake_case，材质名与 ID 一致，配方名对应产物 ID
5. **mapColor 必设**：所有方块注册时必须设置 `.mapColor()`
6. **标签必补全**：挖掘工具标签、材质标签、功能标签在数据生成器中覆盖
