# 重写项目结构说明

## 一、已搭建完成的模块

### 1.1 `init/` — 注册中心

| 文件 | 说明 |
|------|------|
| `ModBlocks.java` | 方块注册，使用 `DeferredRegister<Block>` |
| `ModItems.java` | 物品注册（含 BlockItem、食物、饮品、工具材料），使用 `DeferredRegister<Item>` |
| `ModCreativeModeTabs.java` | 创造模式物品栏 |
| `ModBlockEntities.java` | 方块实体类型注册（claypan、dream_cauldron） |
| `ModBlockEntityRenderer.java` | 方块实体渲染器注册（claypan） |
| `ModFluids.java` | 流体类型注册（16 个流体） |
| `ModMenus.java` | 容器菜单类型注册（claypan、dream_cauldron、mortar） |
| `ModScreens.java` | Screen 注册 |
| `ModNetwork.java` | 网络数据包注册 |
| `ModRecipes.java` | 配方类型注册（claypan、dream_cauldron），mortar 不走配方系统 |
| `ModCommands.java` | 命令注册 |
| `ModFluidContainerRelation.java` | 流体容器关系注册（碗/瓶 → 流体） |

## 四、架构原则

1. **客制化类最小化** — 无特殊机制直接用原版类，仅特殊逻辑放 `world/block/` 或 `world/item/`
2. **注册中心集中** — 方块/物品/菜单等统一在 `init/` 注册
3. **数据生成器驱动** — 所有 JSON 由 datagen 生成，不手动编写
4. **RenderType 统一管理** — cutout / translucent 在 `ClientSetRenderLayer.java` 集中注册
5. **物品标签同步** — `ModItemTagsProvider` 用 `copy()` 从方块标签自动同步
6. **命名规范严格** — ID 语义化 snake_case，材质名与 ID 一致
7. **标签必补全** — 挖掘工具、材质类型、功能标签全覆盖
