# 架构规范

> Forge 1.20.1 模组项目架构规范 — PasterDream Reborn

---

## 包名规范

- **反向域名**：`com.pasterdream.<modid>`
- **全部小写**，不含大写字母
- `modid` 必须与包名中的模组段一致，且满足正则 `[a-z][a-z0-9_]{1,63}`
- 当前 modid：`pasterdream`，包名：`com.pasterdream.pasterdreammod`

## 模块划分

完整目录树见 **[目录结构组织](./目录结构组织.md)**。此处仅说明各模块的职责边界：

| 包 | 职责 | 原则 |
|----|------|------|
| `init/` | 所有注册对象（Blocks、Items、BE、Entities、Menus、Screens、Network、Recipes、Commands、ParticleTypes、Effects、Sounds、Attributes、KeyMappings） | 集中管理，不做业务逻辑 |
| `world/block/` | 有特殊交互的客制化方块类，复杂者独占子包 | 仅特殊逻辑放这里 |
| `world/item/` | 有特殊交互的客制化物品类（如 mortar、drinkandfooditem、dreamnotes、blueprints、curio、StorgeBagItem、ModRarities） | 同上 |
| `world/item/moltengoldtool/` | 炙焰金工具客制化类（剑点燃 + 镐/斧/锹/锄右键急迫） | 共享逻辑提取到 `MoltenGoldToolHelper` |
| `world/item/hellfiretool/` | 狱炎/融骸工具客制化类（剑引燃叠加 + 镐急迫 + 融骸剑熔岩伤害） | 共享逻辑按需提取 |
| `world/item/meltdreamtool/` | 融梦水晶工具客制化类（手持消耗融梦能量修复耐久） | 共享逻辑提取到 `MeltDreamToolHelper` |
| `world/item/armoritem/` | 客制化护甲类（Sculk/Dyedream 套装效果） | 头盔 `inventoryTick` 检测全套触发 buff |
| `world/item/curio/` | 客制化饰品类（33 个 curio 类实现 ICurioItem，含品质提示行 + CurioPassiveHandler 被动处理器） | 品质提示行通过 `ModRarities.qualityTooltip()` 统一生成 |
| `world/entity/` | 实体实现（FoxFire 狐火 + TerraswordWave 极锋剑波） | 实体类放这里，模型/渲染器放 client/ |
| `world/fluid/` + `fluidtype/` | 流体实现与类型定义（16 种流体） | 继承统一的基类 |
| `world/effect/` | 自定义药水效果类（16 个效果：护甲 buff、瞬身术、休憩、烹饪等） | 纯效果定义，业务逻辑放 `world/skill/` |
| `world/skill/` | 玩家技能/能力系统（BlinkSkill 瞬身术位移 + EvasionEffectHandler 回避特效处理器） | 技能逻辑集中管理，按键 → 网络 → 技能 → 效果链路 |
| `world/behavior/` | 方块/物品行为逻辑（树生长器等） | 行为与注册分离 |
| `recipe/` | 通用配方框架 + NBT 保留配方（`NbtPreservingShapelessRecipe`、`NbtPreservingShapedRecipe`） | 升级配方保留附魔 |
| `worldgen/` | ResourceKey + bootstrap 方法（ConfiguredFeature、PlacedFeature、Biome、DimensionType、NoiseSettings、LevelStem） | 不直接注册，由 datagen 产出 JSON |
| `worldgen/feature/` | 自定义 Feature 实现（PinkHugeMushroom、CalciteSpike）+ TreeDecorator | 仅在需要自定义 Feature 时创建 |
| `worldgen/biome/` | 群系注册 + Forge 群系修改器 | 同上 |
| `worldgen/dimension/` | 维度类型、噪声设置、LevelStem 注册 + 维度音乐 | 同上 |
| `worldgen/structures/` | 结构数据生成 Provider | 同上 |
| `client/` | RenderType 注册、HUD 渲染（LoseMind、MeltDreamEnergyTank、SanTank） | `@EventBusSubscriber(value=Dist.CLIENT)` |
| `client/model/` | 实体 GeckoLib 模型（FoxFireModel） | 继承 `GeoModel` |
| `client/renderer/` | 实体渲染器（FoxFireRenderer、TerraswordWaveRenderer） | 继承 `GeoEntityRenderer` 或标准渲染器 |
| `client/particle/` | 粒子行为类 + 粒子提供者注册（11 种粒子） | 每个粒子一个行为类 |
| `datagen/` | 全部 JSON 数据产出（模型、战利品、配方、标签、语言、世界生成） | 唯一的数据来源 |
| `tag/` | 方块/物品标签常量 | 供 datagen 引用 |
| `mixin/` | Mixin 注入（6 个：BlockStateBase、PhantomTarget、GoldenrodTeaBuff、CampfireBlockEntity、SituationalMusic、MusicManager） | 前 4 个为核心 mixin，后 2 个为客户端专属 |
| `network/` | 网络数据包（服务端↔客户端同步） | 每包一功能，复杂功能独占子包 |
| `event/` | Forge 运行时事件处理器（实体伤害、玩家 Tick、生物掉落） | 通过 `MinecraftForge.EVENT_BUS` 注册 |
| `capability/` | Forge Capability 能力系统（MeltDreamEnergy + SAN） | 接口 + 实现 + Provider 模式 |
| `command/` | 调试命令（含 meltdreamenergy、music、san 子命令） | 子命令按功能分子包 |
| `recipe/` | 通用配方框架（接口 + 抽象基类 + Serializer 基类 + MatchResult） | 具体配方类放 `world/block/` 或 `world/item/` |
| `helper/` | 通用工具（带流体槽容器基类、流体容器、渲染、tooltip、属性、原料、配方匹配等） | 不放业务逻辑 |
| `component/` | GUI 可复用组件（按钮、流体槽、只读槽、箭头按钮） | 不依赖具体 Screen |
| `compat/jei/` | JEI 配方兼容（5 种配方类型：claypan、dream_cauldron、mortar、research_table_copy、research_table_research） | 仅使用 JEI API（compileOnly） |

**客制化类规则**：有特殊逻辑（交互、GUI、tick、渲染）的方块/物品类，放在 `world/block/` 或 `world/item/` 包下。无特殊机制的方块/物品直接在注册中心用 `new Block(Properties...)` / `new Item(Properties...)` 注册，不单独创建类文件。

## 注册模式

### DeferredRegister 模式（Forge 标准）

所有注册对象使用 `DeferredRegister` + `RegistryObject` 模式，在 `@Mod` 主类构造方法中统一注册到 Mod 事件总线：

```java
// 1. 在注册中心类中定义 DeferredRegister 和 RegistryObject 常量
public class ModItems {
    public static final DeferredRegister<Item> ITEMS =
            DeferredRegister.create(ForgeRegistries.ITEMS, PasterDreamMod.MOD_ID);
    public static final RegistryObject<Item> EXAMPLE_ITEM =
            ITEMS.register("example_item", () -> new Item(new Item.Properties()));

    public static void register(IEventBus eventBus) {
        ITEMS.register(eventBus);
    }
}

// 2. 在 @Mod 主类构造方法中调用注册
public PasterDreamMod(FMLJavaModLoadingContext context) {
    IEventBus modEventBus = context.getModEventBus();
    ModItems.register(modEventBus);
    ModBlocks.register(modEventBus);
    ModParticleTypes.register(modEventBus);
    ModCreativeModeTabs.register(modEventBus);
}
```

### 注册对象类型对照

| 注册对象 | 注册器类型 | 注册中心类 | 注册表 |
|---------|-----------|-----------|--------|
| 物品 | `DeferredRegister<Item>` | `ModItems` | `ForgeRegistries.ITEMS` |
| 方块 | `DeferredRegister<Block>` | `ModBlocks` | `ForgeRegistries.BLOCKS` |
| 方块实体 | `DeferredRegister<BlockEntityType<?>>` | `ModBlockEntities` | `ForgeRegistries.BLOCK_ENTITY_TYPES` |
| 实体 | `DeferredRegister<EntityType<?>>` | `ModEntities` | `ForgeRegistries.ENTITY_TYPES` |
| 创造模式物品栏 | `DeferredRegister<CreativeModeTab>` | `ModCreativeModeTabs` | `ForgeRegistries.CREATIVE_MODE_TABS` |
| 附魔 | `DeferredRegister<Enchantment>` | `ModEnchantments` | `ForgeRegistries.ENCHANTMENTS` |
| 药水效果 | `DeferredRegister<MobEffect>` | `ModEffects` | `ForgeRegistries.MOB_EFFECTS` |
| 药水 | `DeferredRegister<Potion>` | `ModPotions` | `ForgeRegistries.POTIONS` |
| 粒子类型 | `DeferredRegister<ParticleType<?>>` | `ModParticleTypes` | `ForgeRegistries.PARTICLE_TYPES` |
| 配方序列化器 | `DeferredRegister<RecipeSerializer<?>>` | `ModRecipes` | `ForgeRegistries.RECIPE_SERIALIZERS` |
| 声音事件 | `DeferredRegister<SoundEvent>` | `ModSounds` | `ForgeRegistries.SOUND_EVENTS` |

> 世界生成（`ConfiguredFeature`、`PlacedFeature`、`Biome`、`DimensionType`、`NoiseSettings`、`LevelStem`）不走 `DeferredRegister`，而是通过数据生成器的 `RegistrySetBuilder` + `DatapackBuiltinEntriesProvider` 模式注册。详见下方「世界生成」小节。

## 世界生成

### 注册体系

世界生成相关对象通过数据生成器产出 JSON，而非代码注册。所有 registry bootstrap 类放在 `worldgen/` 包下：

```
worldgen/
├── ModConfiguredFeatures.java       ← ResourceKey + bootstrap（Registries.CONFIGURED_FEATURE）
├── ModPlacedFeatures.java           ← ResourceKey + bootstrap（Registries.PLACED_FEATURE）
├── feature/
│   ├── PinkHugeMushroomFeature.java ← 自定义 Feature 实现
│   ├── CalciteSpikeFeature.java     ← 自定义 Feature 实现
│   └── PinkShroomlightTreeDecorator.java ← 树木装饰器
├── biome/
│   ├── ModBiomes.java               ← ResourceKey + bootstrap（Registries.BIOME）
│   └── ModBiomeModifiers.java       ← ResourceKey + bootstrap（ForgeRegistries.Keys.BIOME_MODIFIERS）
├── dimension/
│   ├── ModDimensionTypes.java       ← ResourceKey + bootstrap（Registries.DIMENSION_TYPE）
│   ├── ModNoiseSettings.java        ← ResourceKey + bootstrap（Registries.NOISE_SETTINGS）
│   ├── ModLevelStems.java           ← ResourceKey + bootstrap（Registries.LEVEL_STEM）
│   └── DyedreamDimensionMusic.java  ← 维度自定义音乐
└── structures/
    ├── ModStructureConfig.java      ← 结构配置常量
    ├── ModStructureProvider.java    ← 结构数据生成 Provider
    ├── ModStructureSetProvider.java ← 结构集数据生成 Provider
    └── ModTemplatePoolProvider.java ← 模板池数据生成 Provider

datagen/
└── ModWorldGenProvider.java         ← DatapackBuiltinEntriesProvider，统一串联所有 bootstrap
```

### 数据流

1. `RegistrySetBuilder` 中按顺序注册各 registry 的 bootstrap（注意依赖顺序：BIOME / DIMENSION_TYPE / NOISE_SETTINGS 在 LEVEL_STEM 之前）
2. `runData` 时 `ModWorldGenProvider` 按 builder 顺序调用各 `bootstrap(context)` 方法
3. `BootstapContext` 的 `register(key, value)` 产出 `data/{modid}/worldgen/` 下的 JSON 文件
4. 运行时通过 `ResourceKey` 引用（如 `ModTreeGrowers` → `ModConfiguredFeatures.DYEDREAM_TREE`）

### 子包划分原则

- `worldgen/biome/` — 群系定义 + Forge Biome Modifier
- `worldgen/dimension/` — 维度类型 + 噪声设置 + LevelStem + DimensionSpecialEffects
- `worldgen/` 根目录 — 地物（ConfiguredFeature + PlacedFeature）
- `worldgen/feature/` — 自定义 `Feature<?>` 实现类 + TreeDecorator，与注册入口类分离
- `worldgen/structures/` — 结构数据生成 Provider

### 维度/群系搬运流程

1. 维度类型 → `ModDimensionTypes.java`（`DimensionType` 构造，含 MonsterSettings）
2. 噪声设置 → `ModNoiseSettings.java`（`NoiseGeneratorSettings` 构造，含 NoiseRouter + SurfaceRules）
3. 维度注册 → `ModLevelStems.java`（`LevelStem` 构造，关联 BiomeSource + ChunkGenerator）
4. 群系注册 → `ModBiomes.java`（`Biome.BiomeBuilder` 构建，含特效、生物生成、雕刻器、特征列表）
5. 如需自定义 DimensionSpecialEffects → `worldgen/dimension/` 下建 `@EventBusSubscriber(Dist.CLIENT)` 类

## 粒子系统

### 注册与实现

粒子分为三层：类型注册、行为实现、群系引用。

```
init/ModParticleTypes.java           ← DeferredRegister<ParticleType<?>> + RegistryObject<SimpleParticleType>

client/particle/
├── LeavesParticle.java              ← 继承 TextureSheetParticle，实现物理行为 + 内部 Provider 类
├── ...（共 11 个粒子行为类）
└── ModParticles.java                ← @EventBusSubscriber(Dist.CLIENT) + RegisterParticleProvidersEvent

assets/pasterdream/
├── particles/<name>.json            ← 纹理映射 JSON
└── textures/particle/<name>.png     ← 粒子纹理
```

### 群系引用

在 `ModBiomes` 的 `BiomeSpecialEffects.Builder` 中通过 `.ambientParticle(new AmbientParticleSettings(particleType, probability))` 引用。客户端自动在群系范围内按概率生成，无需额外 Java 代码驱动。

## 技能系统

瞬身术（Blink）是首个实现的玩家技能系统，其架构链路如下：

```
ModKeyMappings (C 键) → BlinkPacket (net) → BlinkSkill (逻辑) → EvasionEffectHandler (特效)
                                                                  ↓
                                              EvasionBuffEffect + BlinkCooldownEffect (效果标记)
```

### 各层职责

| 层 | 类 | 职责 |
|----|-----|------|
| 按键绑定 | `ModKeyMappings` | 注册 C 键，客户端检测按下 |
| 网络传输 | `skill/BlinkPacket` | 客户端→服务端，携带玩家朝向 |
| 技能核心 | `skill/BlinkSkill` | 条件检查（冷却/消耗/维度/骑乘）、位移计算、冷却设置、buff 施加 |
| 特效处理 | `skill/EvasionEffectHandler` | 粒子爆发、音效播放、冲刺动画、延迟尾迹粒子 |
| 效果标记 | `effect/EvasionBuffEffect` + `effect/BlinkCooldownEffect` | 瞬移后无敌帧 + 冷却计时 |

### 自定义属性

通过 `ModAttributes` 注册三个自定义属性（cd / consume / range），用于控制技能参数。可通过饰品或效果动态修正属性值。

## Mixin 注入

### 使用原则

模组使用 Mixin 进行必要的底层行为注入。当前共 6 个 Mixin 类：

| Mixin | 目标 | Side | 用途 |
|-------|------|------|------|
| `BlockStateBaseMixin` | `BlockStateBase` | Common | 方块状态行为修改 |
| `PhantomTargetMixin` | 幻翼目标选择 | Common | 幻翼目标行为修正 |
| `GoldenrodTeaBuffMixin` | 药水效果系统 | Common | 秋麒麟茶效果处理 |
| `CampfireBlockEntityMixin` | `CampfireBlockEntity` | Common | 营火行为扩展 |
| `SituationalMusicMixin` | 音乐系统 | Client | 情景音乐注入（维度/群系自定义音乐） |
| `MusicManagerMixin` | `MusicManager` | Client | 音乐管理器行为修正 |

### 配置文件

Mixin 配置文件位于 `src/main/resources/pasterdream.mixins.json`，声明所有 Mixin 类及目标环境（client/server）。

## 事件总线体系

Forge 有两级事件总线，按用途选择：

| 总线 | 获取方式 | 用途 | 示例 |
|------|---------|------|------|
| **Mod 总线** | `FMLJavaModLoadingContext.get().getModEventBus()` | 注册、初始化 | `FMLCommonSetupEvent`、注册 DeferredRegister |
| **Forge 总线** | `MinecraftForge.EVENT_BUS` | 游戏运行时事件 | `ServerStartingEvent`、`LivingHurtEvent` |

### 事件订阅方式

1. **构造方法中注册**（适用于 `@Mod` 主类自己的方法）：
   ```java
   modEventBus.addListener(this::commonSetup);
   ```

2. **`@EventBusSubscriber` 注解**（适用于独立类的静态方法）：
   ```java
   @Mod.EventBusSubscriber(modid = PasterDreamMod.MOD_ID)
   public class SomeHandler {
       @SubscribeEvent
       public static void onEvent(SomeEvent event) { ... }
   }
   ```

3. **显式注册到 Forge 总线**（适用于 Mod 总线外的运行时事件）：
   ```java
   MinecraftForge.EVENT_BUS.register(this);
   MinecraftForge.EVENT_BUS.addListener(SomeHandler::onEvent);
   ```

## 客户端分离

- 客户端专属代码放在 `client/` 包下
- 客户端入口类使用 `@Mod.EventBusSubscriber(modid = MOD_ID, value = Dist.CLIENT)` 注解
- 服务端不可用的类（如 `Minecraft`、`EntityRenderer`）**绝不**出现在通用代码路径中

```java
@Mod.EventBusSubscriber(modid = PasterDreamMod.MOD_ID, bus = Mod.EventBusSubscriber.Bus.MOD, value = Dist.CLIENT)
public class ClientModEvents {
    @SubscribeEvent
    public static void onClientSetup(FMLClientSetupEvent event) {
        // 渲染器注册、模型层注册等
    }
}
```

- 使用 Forge `ModConfigSpec` API

## 模组入口类结构

`@Mod` 主类的构造方法是唯一注册入口，按以下顺序组织：

1. 初始化第三方库（如 `GeckoLib.initialize()`）
2. 获取 Mod 事件总线
3. 注册各类 `DeferredRegister`（粒子 → 物品 → 方块 → 物品栏 → 实体 → ...）
4. 注册 Mod 总线监听器（`commonSetup`、`clientSetup`）
5. 注册 Forge 总线监听器（`MinecraftForge.EVENT_BUS.register`）
6. 注册配置文件

## 第三方依赖集成

| 依赖 | 版本 | 用途 |
|------|------|------|
| Curios API | 5.14.1+1.20.1 | 饰品栏扩展（implementation） |
| GeckoLib | 4.8.3 | 实体动画 + 方块动画（implementation） |
| JEI | 15.20.0.112 | 配方查看兼容（implementation） |

> **注意**：所有本地第三方 jar 存放在 `libs/` 目录，通过 `flatDir` 仓库 + `fg.deobf()` 方式引用。`official` 映射下部分第三方 mod 的 refmap 可能不兼容，需修补后使用。

### 映射渠道

项目使用 `official`（Mojang）映射渠道。`official` 映射在开发环境中使用 Mojang 发布的官方反混淆名称，Mixin 框架读取 refmap 的 `mappings` 段进行方法/字段名映射。生产构建时自动混淆为 SRG 名称。

开发环境可能遇到的问题：部分第三方 mod（如 Curios 5.x、GeckoLib 4.x）打包时 refmap 的 `mappings` 段可能不完整或仍包含 SRG 名称，需手动修补（保存在 `libs/` 目录）。

### Curios 集成规范

- 饰品物品放在 `world/item/curio/` 包下，实现 `ICurioItem` 接口
- 插槽注册通过 `@Mod.EventBusSubscriber(bus = Mod.EventBusSubscriber.Bus.MOD)` 注解的 `ModCuriosSlots` 类，在 `InterModEnqueueEvent` 时通过 `InterModComms.sendTo()` 注册
- 插槽标签 JSON 文件手动维护在 `src/main/resources/data/curios/tags/items/` 下
- 可使用 `SlotTypePreset` 预设来创建标准插槽类型，通过 `.size()` 指定插槽数量
- 被动效果统一通过 `CurioPassiveHandler` 处理，每 tick 检测装备状态

### GeckoLib 集成规范

- 动画实体/方块使用 `GeoEntity` / `GeoBlockEntity` 接口
- 渲染器继承 `GeoEntityRenderer` / `GeoBlockEntityRenderer`
- 模型类继承 `GeoModel`，动画控制器继承 `AbstractAnimationController` 或使用 builtin 动画
- 几何模型文件（`.geo.json`）放在 `assets/pasterdream/geo/`
- 动画文件（`.animation.json`）放在 `assets/pasterdream/animations/`
- 显示设置放在 `assets/pasterdream/models/displaysettings/`
- 特殊模型（粒子/展示用）放在 `assets/pasterdream/models/custom/`

### JEI 集成规范

- JEI 兼容代码全部放在 `compat/jei/` 包下
- 使用 `@JeiPlugin` 注解注册插件
- 仅使用 JEI API（`compileOnly` 依赖确保不直接引用内部实现）
- 当前支持 5 种配方类型：claypan、dream_cauldron、mortar、research_table_copy、research_table_research
