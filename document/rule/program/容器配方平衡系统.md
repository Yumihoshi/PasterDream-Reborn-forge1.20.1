# 容器配方平衡系统

## 背景

模组中存在带 `craftRemainder` 的填充容器物品（如 `GLASS_JAR_OF_WATER` → `GLASS_JAR`）。当这类物品作为合成原料时，Forge 默认会在合成台中返还对应的空容器。

当配方产物**也是**填充容器时，会出现容器数量不平衡——原料返还空容器 + 产物也是容器 = 凭空多出容器。

## 核心原理

系统通过覆盖 `getRemainingItems()` 实现自动配平：

- **E** = 配方原料中空容器的数量（物品本身即为容器类型）
- **F** = 配方原料中填充容器的数量（`craftRemainder` 返回该容器类型）
- **O** = 产物中填充容器的数量（产物数量）

平衡公式：**cancelCount = min(F, max(0, O - E))**

- `O > E + F`：**无法配平**，datagen 直接报错中断
- `O = E`：天然平衡，无需干预
- `O > E`：取消 `O - E` 个填充原料的 remainder，用产物的容器替代
- `O < E`：空容器过剩，remainder 正常返还（不会凭空消失容器）

## 容器类型检测

系统通过两级检测判断物品是否为填充容器：

1. **craftRemainder**（优先）：`Item.getCraftingRemainingItem(stack)` 返回非空 → 该返回物品即为空容器类型
2. **FluidContainerRegistry**（回退）：`FluidContainerRegistry.getEntryForFillToEmpty(item)` 返回非 null → `entry.emptyItem` 即为空容器类型

## 使用方法

### 新建填充容器物品时必须做的事

1. **设置 craftRemainder**：

```java
// PasterDreamDrinkAndFoodProperties 已添加 craftRemainder 协变返回，无需强转
public static final RegistryObject<Item> GLASS_JAR_OF_XXX = ITEMS.register("glass_jar_of_xxx",
    () -> new PasterDreamDrinkItem(new PasterDreamDrinkAndFoodProperties()
        .food(...)
        .craftRemainder(ModItems.GLASS_JAR.get())));

// 普通 Item
public static final RegistryObject<Item> GLASS_JAR_OF_XXX = ITEMS.register("glass_jar_of_xxx",
    () -> new Item(new Item.Properties().craftRemainder(ModItems.GLASS_JAR.get())));
```

2. **注册 FluidContainerRelation**（在 `ModFluidContainerRelation.java` 中）：

```java
FluidContainerRegistry.register(
    ModItems.GLASS_JAR.get(),           // 空容器
    Fluids.WATER,                       // 流体
    125,                                // 量 (mB)
    ModItems.GLASS_JAR_OF_WATER.get()   // 填充容器
);
```

### 在 Datagen 中编写配方

产物是填充容器的配方，使用 `saveContainerBalancedShapeless`（无序）或 `saveContainerBalancedShaped`（有序）：

```java
saveContainerBalancedShapeless(
    ShapelessRecipeBuilder.shapeless(RecipeCategory.MISC, ModItems.GLASS_JAR_OF_DREAM_JUICE.get(), 1)
            .requires(ModItems.GLASS_JAR_OF_WATER.get(), 1)
            .requires(Items.BLUE_DYE, 1)
            .requires(Items.PINK_DYE, 1)
            .unlockedBy(...),
    pWriter, "glass_jar_of_dream_juice",
    // 校验参数：(产物, 产物数量, 原料1, 原料数量1, ...)
    ModItems.GLASS_JAR_OF_DREAM_JUICE.get(), 1,
    ModItems.GLASS_JAR_OF_WATER.get(), 1,
    Items.BLUE_DYE, 1,
    Items.PINK_DYE, 1);
```

参数说明：
- 前 3 个固定：builder, writer, recipeName
- 之后是 `(产物, 产物数量)` + 多组 `(原料, 原料数量)` —— 同时用于校验和生成配方
- **不需要**再手动添加空罐子来配平，系统自动抵消多余的 remainder

### 不需要改动的配方

以下情况不需要使用容器平衡配方类型：
- 产物不是填充容器（如面团、蛋糕等）
- 配方中填充容器原料只是为了正常返还空容器

## 常见问题

**Q: 配方已有空罐子作为原料，还需要改吗？**
A: 如果空罐子数量恰好等于产物的填充罐子数量（E = O），天然平衡，可以不改。但建议改成 `saveContainerBalancedShapeless`，防止未来有人修改配方时引入不平衡。

**Q: 如果配方用到了多种不同类型的容器（如桶+罐子）？**
A: 系统按容器类型分别计算，只取消同类容器的 remainder。桶的 remainder 不会被罐子的配平逻辑影响。

**Q: 不在 datagen 中生成的手写 JSON 配方怎么办？**
A: 使用 `"type": "pasterdream:nbt_preserving_shapeless"` 或 `"nbt_preserving_shaped"` 作为配方类型，运行时自动配平。但无法享受 datagen 阶段的静态校验，需要手动确保配平或等待运行时 WARN 日志。
