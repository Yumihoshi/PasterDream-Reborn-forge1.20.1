# 目录结构组织

> Forge 1.20.1 模组项目标准目录结构 — PasterDream Reborn

---

## 项目根目录

```
PasterDream-Reborn-forge1.20.1/
├── build.gradle              # Gradle 构建脚本（模组依赖、运行配置、数据生成）
├── settings.gradle           # Gradle 设置（插件仓库、工具链）
├── gradle.properties         # 构建属性（MC 版本、模组 ID、版本号等）
├── gradle/                   # Gradle Wrapper
├── gradlew / gradlew.bat     # Gradle 启动脚本
├── changelog.txt
├── CREDITS.txt
├── LICENSE.txt
├── README.txt
├── libs/                     # 第三方本地 jar（Curios、GeckoLib、JEI 等，部分需修补 refmap）
├── reference/                # 旧版模组源码参考（net.pasterdream），不参与编译
├── document/                 # 项目文档（非代码）
│   ├── rule/
│   │   └── program/          # 程序规范文档
│   └── design/               # 策划与设计文档
│       ├── 策划文档.md
│       ├── 命名规范文档.md
│       └── ID映射表.md
└── run/                      # 开发环境运行目录
```

## 关键目录说明

| 目录 | 说明 | 手动/自动 |
|------|------|----------|
| `src/main/java/` | Java 源代码 | 手动编写 |
| `src/main/resources/` | 静态资源（纹理、手动 JSON、Mixin 配置） | 手动维护 |
| `src/generated/resources/` | 数据生成器输出（blockstate、model、lang、loot_table、tags） | 自动生成（runData），**不可手动编辑** |
| `libs/` | 第三方本地 jar（Curios、GeckoLib、JEI） | 手动放置，需修补 refmap |
| `document/` | 项目文档（规范、设计笔记等） | 手动维护 |
| `reference/` | 旧版模组源码参考 | 只读，不参与编译 |

## 包名规范

```
com.pasterdream.pasterdreammod
```

- 全部小写，不含大写字母
- `modid = pasterdream` 满足正则 `[a-z][a-z0-9_]{1,63}`
- 与 `gradle.properties` 中的 `mod_id` 保持一致
