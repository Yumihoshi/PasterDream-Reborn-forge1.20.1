# 命名规范文档

> PasterDream Reborn —— 配方、材质及其他资源命名规范

---

## 一、配方命名规范

### 1.1 问题描述

源模组中的配方文件名大量使用 `crafting_` + 数字编号的格式，完全不具有可读性：

```
crafting_0.json
crafting_1.json
crafting_2.json
...
crafting_156.json
```

无法从文件名判断该配方的产出物品，给维护带来困难。

### 1.2 规范

配方文件名应与产出物品的 ID 一致（或加类型前缀区分）：

```diff
- crafting_42.json
+ dream_shard.json

- crafting_103.json  
+ dyedream_quartz_block.json
```

**规则**：

| 配方类型 | 命名格式 | 示例 |
|---------|---------|------|
| 工作台（有序/无序） | `{产物id}.json` | `dyedream_quartz_block.json` |
| 熔炉/高炉 | `{产物id}_smelting.json` | `titanium_ingot_smelting.json` |
| 锻造台 | `{产物id}_smithing.json` | `shadow_sword_smithing.json` |
| 切石机 | `{产物id}_stonecutting.json` | `cyan_stone_brick_stonecutting.json` |

**特殊情况**：

- 同一产物有多种合成方式时，加来源描述：`{产物id}_from_{来源}.json`
  - 例：`dyedream_quartz_block_from_dyedream_quartz.json` 与 `dyedream_quartz_block_from_stonecutting.json`

## 二、材质命名规范

### 2.1 问题描述

源模组中的材质文件大多使用中文拼音命名，与方块/物品的注册 ID 完全脱节：

```
tuzhanggui.png          → 土长官？意图不明
zhuomianshi.png         → 后面石？含义模糊
molishao.png            → 魔力哨？无法对应到具体物品
```

这类命名不仅难以维护，而且在搬运时需要逐一对照确认，效率极低。

### 2.2 规范

材质文件名必须与方块/物品的注册 ID 保持对应：

```
材质 → assets/pasterdream/textures/{type}/{item_or_block_id}.png
```

**规则**：

| 资源类型 | 路径格式 | 示例 |
|---------|---------|------|
| 方块纹理 | `textures/block/{block_id}.png` | `textures/block/dyedream_dirt.png` |
| 物品纹理 | `textures/item/{item_id}.png` | `textures/item/dream_shard.png` |
| 实体纹理 | `textures/entity/{entity_id}.png` | `textures/entity/golden_fox.png` |
| GUI 纹理 | `textures/gui/{gui_name}.png` | `textures/gui/dream_cauldron.png` |

**变体处理**：同一方块/物品有多个纹理变体时，加后缀区分：

```
textures/block/cyan_stone_brick.png
textures/block/cyan_stone_brick_cracked.png
textures/block/cyan_stone_brick_chiseled.png
```

**多方块纹理**：多面不同的方块，加面描述：

```
textures/block/dream_cauldron_top.png
textures/block/dream_cauldron_bottom.png
textures/block/dream_cauldron_side.png
```

### 2.3 材质格式

- **格式**：PNG
- **分辨率**：默认 16×16（如需高清，统一使用 32×32 或 64×64 并声明）
- **色彩空间**：sRGB
- **透明处理**：如需透明度，使用 PNG alpha 通道

## 三、其他资源命名规范

### 3.1 模型文件

由数据生成器自动产出，命名与注册 ID 一致，无需手动管理。

### 3.2 语言文件

由数据生成器自动产出。如下手动补充：

- 英文：`en_us.json`（`assets/pasterdream/lang/`）
- 中文：`zh_cn.json`（`assets/pasterdream/lang/`）

### 3.3 进度

格式：`{namespace}/{advancement_id}.json`，放在 `data/pasterdream/advancements/` 下。

### 3.4 战利品表

格式：`{type}/{loot_table_id}.json`，放在 `data/pasterdream/loot_tables/` 下。由数据生成器自动产出。

### 3.5 标签

格式：`{type}/{tag_name}.json`，放在 `data/{namespace}/tags/` 下。由数据生成器自动产出。

---

## 四、命名检查清单

搬运每个方块/物品时，对照检查：

- [ ] 注册 ID 是否语义化、snake_case、全小写？
- [ ] 配方文件名是否与产物 ID 对应？（如适用）
- [ ] 材质文件名是否与注册 ID 对应？
- [ ] 材质格式是否符合规范（PNG、分辨率、sRGB）？
- [ ] 纹理路径是否放在正确的子目录下（`block/`、`item/`、`entity/`）？
